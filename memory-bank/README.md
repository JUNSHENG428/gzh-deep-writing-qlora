# Memory Bank 使用说明

本目录是项目的「外部记忆」，供 Agent 在上下文丢失/新会话开始时快速恢复状态。

## 阅读顺序（新会话必读）
1. `01_project_brief.md` —— 项目是什么、目标、硬件约束（很少变动）
2. `02_pipeline.md` —— 数据管线各阶段与脚本对照表（脚本变了才更新）
3. `03_progress.md` —— **当前进度 + 下一步行动**（最常更新，动手前先看这个）
4. `04_decisions.md` —— 关键技术决策及理由（做过的选择别重新纠结）

## 维护规则
- 每完成一个阶段性任务（跑完一个脚本、做完一个决策），更新 `03_progress.md`
- 做出新的技术选型时，追加到 `04_decisions.md`（附日期和理由）
- 更新时修改「最后更新」时间戳
